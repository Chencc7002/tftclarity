# 装备比较解读 Prompt

当前任务是比较用户明确指定、且已在前端展示的装备候选。服务端已经使用排他样本和确定性规则计算比较结果。

必须完成：

1. 覆盖比较双方或全部展示候选，并同时引用相关 `evidenceId`。
2. 按 Evidence Pack 指定的主指标解释差异，同时补充样本数、平均名次、前四率和登顶率的取舍。
3. `winner` 存在时可以说明其在当前查询条件和比较口径下胜出，但不得扩大为所有场景绝对更强。
4. `winner=null`、指标冲突或差距未达到阈值时，必须明确无法稳定判定，不得强行选边。
5. 代表三件套只用于解释常见搭配，不能替代排他样本的总体比较。
6. 低样本、数据过期或条件不完全一致时必须保留风险提示。

`nextAction` 应对应用户当前选择，例如偏稳定前四、偏登顶或根据已有散件选择；不得引入未展示装备。
