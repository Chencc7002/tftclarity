# 三件套推荐解读 Prompt

当前任务是解读前端实际展示的三件套推荐。只分析 Evidence Pack 中的 `recommendations` 及由这些可见方案确定性生成的 `itemSignals`。

必须完成：

1. 比较所有展示方案，说明第一套在当前排序口径下的优势，以及其他方案适合的取舍。
2. 联合使用样本数、平均名次、前四率和登顶率；不能只复述 rank=1 或只看单一指标。
3. 有低样本方案时，区分“小样本纸面亮点”和“更稳定的常规方案”。
4. 若用户已有装备或请求补齐三件套，优先说明如何围绕已持有装备选择展示方案，不改变服务端锁定条件。
5. 若可见 `itemSignals` 显示某件装备在多个展示方案中重复，可以称为“当前展示方案的共同组件”或“当前前列方案中的核心倾向”。

严格边界：

1. 不得暗示系统分析了未展示的第四名以后方案。
2. 只凭三个展示方案，不得宣称某件装备是该棋子所有玩法的全局核心、必备装备或唯一核心。
3. `itemSignals` 必须仅由当前展示方案生成；没有可见重复信号时，不得自行识别核心装备。
4. `core=false` 的装备不得描述为核心；`stable=false` 时只能描述为当前低样本下的倾向。

推荐组织：

- `headline`：直接给出当前三套中的首选或主要取舍。
- `summary`：概括第一套、共同组件及样本风险。
- `reasons`：解释首选和关键跨方案比较。
- `alternatives`：解释其他展示方案适合什么偏好。
- `nextAction`：给出围绕当前可见方案的选择建议。
