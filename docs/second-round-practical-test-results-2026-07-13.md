# 第二轮实操测试结果（2026-07-13）

## 1. 结论

- 测试分支：`codex/comps`
- 对照文档：`docs/second-round-practical-test-cases.md`
- P0 冒烟：**2/12 通过，10/12 失败，当前不满足发布条件**。
- 全部 126 条：**28 通过、13 部分通过、78 失败、4 未执行、2 需求冲突、1 不适用**。
- 当前确认已生效：阵容榜“大师以上呢？”的意图继承、明确指定纹章后的锁定补装、移除样本下限、特殊装备查询默认样本下限为 0、排除指定纹章、会话清空隔离、窄屏无横向溢出。
- 当前主要缺口：装备百科接口与页面、光明装/纹章的单装备详情查询、泛称纹章澄清、七种观星者精确名称、`巨九` 别名、完整装备对比语义与统计门禁、装备目录审核页。
- **分支核对更正：上述装备百科、`正义`/`合剂`、`巨九`、comparison 和目录审核实现并非从未开发。** `c766179 feat: add official item encyclopedia` 与 `694ecd7 feat: add conversational item comparison and catalog audit` 已存在于 `codex/item-comparison`，但均不是当前 `codex/comps` HEAD 的祖先。本报告的失败结果描述的是当前分支实际行为；修复动作应优先集成已有分支实现并解决与当前 comps/UI 代码的冲突。
- `STAR-09`、`STAR-10` 与最新需求“删除阵容自动补全；只有玩家自行输入阵容条件才携带”冲突，本报告标记为“需求冲突”，不作为当前实现失败。

状态定义：

- **通过**：本轮实际执行或现有离线测试能够完整证明预期。
- **部分通过**：安全边界或部分语义正确，但完整输出/集成行为不满足预期。
- **失败**：实际行为与预期不符，或所需功能不存在。
- **未执行**：当前环境缺少执行依赖或本轮没有可靠夹具，不能以静态推断冒充实操结果。
- **需求冲突**：测试文档与用户最新明确需求相反，应先更新测试文档。
- **不适用**：前置功能不存在，文档也允许记为不适用。
