> 视觉依据：本次确认的「Set 17 · 星神」效果图；`C:\Users\Chencc\Downloads\gemini-code-1783831887687.html` prototype。

## 1. 目标与边界

### 1.1 产品目标

TFTAgent 是云顶之弈局内数据检索助手。用户通过自然语言提问，获得以下任一种结果：

- 指定英雄的三件套推荐、已持有装备后的补齐推荐，或装备对比。
- 指定英雄的单件装备排行及常见搭配。
- 当前版本阵容排行榜。
- 对英雄、装备或羁绊的澄清候选。

界面要把“对话上下文”和“可快速读取的数据结论”同时保留下来，适合在游戏窗口旁使用，也适合放大后作为桌面查询工具。

### 1.2 非目标

- 不做 MetaTFT 式的多级导航站点、玩家战绩主页或广告位。
- 不做棋盘识别、OCR、游戏内存读取、自动操作游戏。
- 不把 LLM 的文字推理作为数据指标或最终排名来源。

### 1.3 设计原则

1. **结论优先**：先显示推荐结论，再显示统计、条件与来源。
2. **对话不丢失**：任何当前结果均可追溯到对应用户提问与已继承条件。
3. **数据可解释**：必须展示样本、统计口径、数据源、更新时间和缓存/过期风险。
4. **尺寸无关**：同一功能在小窗、普通浮窗、最大化窗口均可完成；不允许横向溢出。
5. **局内克制**：星神主题只服务于层级与氛围，不能降低文字、图标与指标可读性。

## 3. 视觉系统

### 3.1 赛季主题：Set 17「星神」

整体采用“深色游戏桌面背景上的浅色星神应用窗口”：应用本体以暖白和半透明白色为阅读底，顶部标题栏使用青蓝到蓝紫的星云渐变。星尘、星座线和柔和辉光只能作为低对比度纹理出现。

| `--panel` | `#FFFFFF` | 卡片、输入框、抽屉 |
| `--ink` | `#253042` | 主文字 |
| `--muted` | `#748096` | 次级文字、元信息 |
| `--line` | `#E5E9F0` | 分割线、普通描边 |
| `--star-cyan` | `#3EAEEB` | 主交互、当前输入来源 |
| `--star-indigo` | `#5678E8` | 焦点、语言控件选中态 |

| `--warning` | `#E99536` | 低样本、过期风险 |

### 3.2 字体与图标

- 中文：`Microsoft YaHei` 或系统无衬线；英文：`Segoe UI` / `Inter`。
- 正文最低 13px；数据主值 20–24px；说明与来源最低 11px。
- 英雄、装备、羁绊一律优先使用现有 asset manifest 图标；加载失败时使用固定尺寸文字占位，不得改变布局。
- 顶栏品牌图标为“六角星芒”而非 Riot 或 MetaTFT 标志。

### 3.3 组件状态

- 主按钮：青蓝填充、白字；hover 加深，focus 有 2px 星光外描边。
- 普通按钮：白底细描边；危险操作必须先确认或提供可撤回提示。
- 卡片：8–14px 圆角，细描边，轻阴影；最佳卡只使用金色描边和标签，不使用大面积高饱和黄色。
- 支持 `hover`、`focus-visible`、`disabled`、`loading`、`empty`、`error`、`stale` 状态。

## 4. 应用窗口与响应式规范

### 4.1 桌面窗口

应用是可置顶的桌面浮窗，而非固定 460px 宽网页：

- 用户可拖动标题栏移动窗口；拖动四边和四角任意改变宽高。
- 标题栏提供最小化、最大化/还原、关闭；最大化后占满可用桌面区域。
- 右下角显示细小缩放握柄；原生壳提供 resize cursor 与真正的窗口 resize，前端不得用 CSS 假装改变系统窗口大小。
- 建议窗口最小尺寸：`360 × 560px`；建议默认尺寸：`1200 × 760px`；最大尺寸由操作系统工作区决定。
- 若浏览器 app window 在目标发布环境不能提供自由拖拽 resize，需在桌面封装层（如 WebView2、Tauri、Electron 或等效宿主）实现；前端只监听 `ResizeObserver`/viewport 改变并重排。

### 4.2 布局断点

| 宽度 | 布局 | 关键规则 |
| --- | --- | --- |
| `>= 1100px` | 三列：对话 / 结果 / 设置（设置可关闭） | 对话约 36–40%，结果弹性填充，设置 260–300px。 |
| `760–1099px` | 两列：对话 / 结果；设置为覆盖抽屉 | 中间分隔线允许拖动；结果列不小于 360px。 |
| `520–759px` | 单列主工作流 | 当前助手结果置于对话中的独立结果块；设置为全高抽屉。 |
| `360–519px` | 紧凑小窗单列 | 统计变为 2×2；文本按钮缩短或仅显示图标+tooltip；备选方案默认折叠。 |

高度规则：顶部标题栏固定，输入区固定在底部；对话区和结果区各自独立滚动；高度不足时优先减少卡片间距、折叠次级信息，不能遮挡输入框。

### 4.3 列宽与分隔线

- 对话列初始宽度为可用主体宽度的 38%，最小 320px、最大 520px。
- 对话/结果之间使用 1px 分隔线和居中的拖动把手；`cursor: col-resize`。
- 用户改变分隔位置后以本地偏好保存比例；窗口过窄时自动撤销为单列，不保留不可用的分栏状态。
- 设置列打开时不挤压结果到不可读宽度：空间不足时必须切换为右侧覆盖抽屉。

## 5. 信息架构与页面结构

```text
DesktopWindow
├─ TitleBar
│  ├─ Brand（星芒图标、TFTAgent、Set 17 · 星神）
│  ├─ LanguageToggle（中文 / EN）
│  ├─ RuntimeBadge（就绪、查询中、缓存、过期缓存、错误）
│  └─ WindowControls
├─ Workspace
│  ├─ ConversationPane
│  │  ├─ MessageList
│  │  └─ Composer（固定底部）
│  ├─ ColumnResizer
│  ├─ ResultPane
│  │  └─ ResultView（按返回类型切换模板）
│  └─ SettingsPanel（宽屏常驻可选；其余宽度为抽屉）
└─ ResizeGrip
```

### 5.1 标题栏

左侧显示品牌、赛季标签；右侧依次为语言切换、可选官网链接、设置、运行状态、窗口控制。

- 语言控件采用分段按钮：`中文 | EN`，当前项有实心/半透明高亮。点击立即切换 UI 文案，不重置会话、不重新请求数据。
- 英雄、装备、羁绊名称按当前语言显示；查询原文、API 名和已产生的历史消息保持原始内容。
- 运行状态只展示一个主状态：`就绪`、`查询中`、`实时`、`本地缓存`、`过期缓存`、`失败`。详细数据在设置中展示。

### 5.2 对话列

对话列是操作入口和上下文记录，不重复承担大数据卡展示。

- 用户消息右对齐，浅蓝气泡；助手消息左对齐，星芒头像。
- 助手消息包含：简短结论、解释文本、条件标签、条件来源图例，以及“在结果区查看详情”的跳转/高亮联动。
- 连续追问需展示时间和消息顺序；用户如问“如果只有两件成装呢？”必须保留上一轮英雄、羁绊等继承条件。
- 加载态依次显示：`理解条件 → 查询数据 → 计算排名`，停止后明确显示已停止。
- 澄清态显示问题、候选项、`查询` 与 `保存候选` 操作；错误态提供重试入口。

### 5.3 输入区（Composer）

- 始终固定在对话列底部；输入框支持 Shift+Enter 换行、Enter 发送。
- 占位文案：中文 `问问霞的装备，Enter 发送…`；英文 `Ask about Xayah's items. Press Enter to send…`。
- 操作：重试上一条、刷新数据、清空会话、发送；请求中显示停止按钮并禁用重复发送。
- 加号仅用于后续扩展快捷建议；本期不得暗示已支持截图上传或 OCR。

### 5.4 结果列

结果列永远展示**最近一次成功或正在查询的结果**。选择历史消息时，应允许把相应结果恢复为当前结果；本期可先以最新结果为准。

通用顶部：结果类型标题、信息提示图标、当前查询摘要、刷新状态。

#### A. 三件套推荐 / 补齐 / 对比

1. **最佳推荐卡**：标题、`最佳`标签、三个装备图标与名称、锁定装备状态。
2. **核心统计**：前四率、吃鸡率、平均名次、样本数；宽屏 4 列，紧凑窗 2×2。
3. **备选方案**：最多展示两个稳定备选；低样本结果使用显式“低样本参考”标签，不能称为最佳。
4. **差异说明**：有已持有、排除或比较条件时，显示补齐、替换、前四率差值与样本差值。
5. **反馈**：每个推荐卡有“有帮助 / 没帮助”按钮；提交后显示已记录且避免重复提交。

#### B. 单件装备排行

- 列表项含装备图标、装备名、排名、四项统计、常见搭配与重复件信息。
- 返回为空时，展示解释、当前条件与建议修改项，不使用空白页面。

#### C. 阵容排行榜

- 顶部说明统计天数、样本阈值、段位、刷新时间与缓存状态。
- 分区展示前四率、吃鸡率、平均名次、热门度等榜单；每项可展开。
- 展开项展示核心英雄、装备、羁绊、完整统计和来源；低样本只出现在“低样本参考”区，不进入主排名。

#### D. 澄清、空结果、错误

- 澄清：明确指出缺少或无法确定的实体，提供候选按钮。
- 空结果：解释是条件过严、装备不可用、无稳定默认阵容或数据暂无；展示可修改的条件标签。
- 错误：不暴露敏感配置，提供重试和刷新；保留用户输入。

### 5.5 条件、来源与风险

助手消息和结果区至少有一处完整显示以下信息：

- 条件 chips：英雄、星级、羁绊、装备范围、已携带、已排除、比较项、天数、段位、样本阈值、排序、默认 Comp。
- 条件来源图例：本次输入（蓝）、会话继承（绿）、用户偏好（橙）、系统默认（金）。
- 数据来源：provider、endpoint、更新时间、实时/缓存/过期缓存。
- 风险：外部数据、过期缓存、低样本、默认 Comp 降级等。

这些信息不可仅存在于“内部响应”调试区。

### 5.6 设置面板

设置不是主导航。宽屏可作为右侧 260–300px 列；中小窗口为带遮罩的右侧抽屉。

1. **高级查询默认值**
   - 样本阈值：10 / 50 / 100 / 500 / 1000。
   - 装备范围：普通、光明、神器、特殊。
   - 排序：前四优先、吃鸡优先、稳健优先。
   - 统计天数、默认 Comp 策略、结构化解析策略、段位多选。
2. **数据与运行状态**
   - 数据源、数据表/endpoint、刷新时间、缓存状态、LLM 状态。
3. **别名审核（高级）**
   - 筛选、启用/停用、批量审核、导出、下载、清候选。
4. **危险/恢复操作**
   - 清历史、恢复默认设置；清理动作使用确认提示。

## 6. 交互与状态规范

| 交互 | 行为 | 验收标准 |
| --- | --- | --- |
| 发送查询 | 追加用户消息，创建助手加载消息，请求 `/api/recommend` | Enter 发送、Shift+Enter 换行；请求中可停止。 |
| 刷新 | 以最近一次输入重新请求并要求刷新数据 | 不丢失会话；按钮在无历史输入时禁用。 |
| 清空 | 清理服务端会话并创建新 conversationId | 清空后回到欢迎消息，输入框可直接使用。 |
| 点击条件标签 | 将对应“修改条件”提示写入输入框 | 不自动发请求，用户可编辑后发送。 |
| 点击候选 | 将候选写入输入并查询；或保存为待审核别名 | 候选保存成功有明确反馈。 |
| 切换语言 | 更新前端文案和展示名称 | 不清空历史、不更改 API 参数和偏好值。 |
| 拖动中间分隔线 | 实时调整左右列宽，持久化比例 | 小于双列阈值时自动单列，恢复宽度后按记忆比例还原。 |
| 拖动窗口边缘 | 由桌面壳调整窗口；前端响应 resize | 所有断点无横向滚动和无遮挡控件。 |
| 打开设置 | 宽屏显示侧列/中小屏显示抽屉 | ESC、关闭按钮和点击遮罩均可关闭抽屉。 |

## 7. 本地化规范

新增 `i18n` 层，推荐以 `zh-CN`、`en-US` 两套资源文件实现。

- 静态 UI 文案必须从资源表读取：按钮、标题、空态、错误、状态、设置字段、图例、tooltip。
- API 返回的英雄/装备/羁绊应由当前 catalog 提供当前语言名称；缺失翻译时退回 canonical 名称。
- 数字格式由 `Intl.NumberFormat(locale)` 处理；百分比、样本数、日期/相对时间随语言变化。
- 用户输入和历史原文不做机器翻译。
- 保存语言偏好到长期 preferences；默认 `zh-CN`。

## 8. 可访问性与性能

- 所有图标按钮必须有本地化 `aria-label` 与 tooltip。
- 键盘：Tab 顺序符合阅读顺序；Enter/Shift+Enter、Escape 关闭抽屉；可通过键盘操作分段控件和反馈按钮。
- 颜色不能是唯一信息来源：低样本、缓存和条件来源均需文字或图标辅助。
- 不得在主线程做大规模图标解码；图片使用固定尺寸、懒加载与占位。
- 每个滚动区只保留一个滚动职责，避免页面、对话和结果发生滚动冲突。

## 10. 开发验收清单

### 功能验收

- [ ] 可完整呈现推荐、单件装备排行、阵容排行、澄清、空结果、错误、加载、低样本与过期缓存。
- [ ] 所有已有偏好项可在设置中读取、修改、保存和恢复默认。
- [ ] 有帮助/没帮助、候选查询/保存、重试、刷新、清空行为与当前接口一致。
- [ ] 中英文切换覆盖全部静态 UI 文案，且不丢会话。
- [ ] 用户可改变窗口宽高；窗口状态随断点正确重排。

### 视觉验收

- [ ] 1440×900 最大化、1200×760 普通浮窗、760×700、520×700、360×560 五种尺寸均通过截图检查。
- [ ] 无任何水平滚动条；装备名、样本数、条件 chips、来源文本不被裁切。
- [ ] 360px 宽度下，统计为 2×2，输入区始终可见，设置以抽屉呈现。
- [ ] 主题纹理不干扰文字对比度；主要正文满足 WCAG AA 对比度目标。
- [ ] 图标缺失时，固定占位不造成卡片跳动。

### 数据验收

- [ ] 结果中展示的指标与 API 返回的数据一致；前端不自行篡改排名。
- [ ] 低样本和缓存风险总有显式文字提示。
