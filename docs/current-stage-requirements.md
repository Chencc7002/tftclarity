# TFTAgent 当前阶段需求基线

更新时间：2026-07-11

本文档冻结当前代码已经实现的产品能力、技术边界和验收口径。阵容与装备指标都必须来自结构化名次分布，LLM 不得代替数据查询、本地统计或排序。

## 1. 产品定位

TFTAgent 是《云顶之弈》局内数据检索小窗。当前阶段的核心产品是：

```text
单英雄三件套装备检索
+ 全局阵容分指标排行榜
+ 中文自然语言条件解析
+ MetaTFT 统计查询
+ 本地过滤、计算和排序
+ 图标化可展开小窗展示
```

它不是让模型凭游戏知识作答的聊天机器人。数据指标必须来自 API 返回的结构化统计，LLM 只承担可选的自然语言结构化解析。

## 2. 当前阶段目标

玩家可以输入明确英雄装备条件，也可以直接查询当前版本阵容榜。系统应完成：

1. 识别英雄中文名、英文名、称号、常用俗称、拼音和繁体写法。
2. 解析星级、羁绊、装备数量、装备类型、已持有装备、排除装备、排序和样本阈值。
3. 条件缺失时补全 2 星、3 件普通装备、当前版本、近 3 天、默认段位和样本阈值。
4. 使用 MetaTFT `/comps` 为该英雄选择主流阵容/羁绊上下文。
5. 查询 Explorer `unit_builds/{unit}` 获取 `placement_count`。
6. 本地计算样本数、前四率、登顶率和平均名次。
7. 过滤当前版本不可用、历史和不符合装备策略的装备。
8. 返回 1 个推荐和最多 2 个备选，并透明展示查询条件和数据来源。
9. 对阵容榜使用 `exact_units_traits2` 的真实八桶 `placement_count`，按前四、登顶、均名和热门分别排序。
10. 使用稳定 cluster 或阵容指纹去重，异常与低样本形态透明过滤/降级，并以英雄、装备和羁绊图标展示构筑。

典型输入：

```text
霞带哪三件装备最好？
努努适合带什么装备？
霞已经有羊刀，剩下两件怎么带？
霞不要轻语，其他装备怎么带？
2星霞3观星，羊刀和无尽哪个更好？
当前版本最强阵容有哪些？
最热门的阵容。
```

## 3. 已实现功能

### 3.1 实体与条件解析

- 当前赛季英雄、羁绊和装备目录可从 Explorer 与 `/comps` 动态生成并按 patch 缓存。
- 高置信中文名和俗称由本地 override 维护；当前审计覆盖英雄 62/62、羁绊 100/100、装备入口 169/169。
- 支持 NFKC、常见繁简转换、无声调拼音和中英文混合输入。
- 多英雄、实体别名碰撞、排序冲突、缺少比较项和显式低置信实体会在远程请求前阻断并澄清。
- BM25、编辑距离和本地稀疏 TF-IDF 只生成候选；未经人工确认的低置信别名不会进入主目录。

### 3.2 单英雄装备检索

- 支持 1/2/3 星，默认 2 星。
- 支持普通、光明、神器、纹章和特殊装备范围。
- 支持已持有装备锁定、显式装备排除和两件候选装备对比。
- 支持前四优先、登顶优先和稳健排序。
- 低样本结果保留统计，但降级为“低样本参考”，不宣称稳定推荐。
- 当前版本不可用装备由本地显式规则裁决，不因第三方 API 是否返回 token 自动改变。

### 3.3 默认阵容上下文

- 已接入 MetaTFT `latest_cluster_info`、`comp_options` 和 `comp_builds`。
- `/comps` 当前只用于给单英雄装备查询选择主流 cluster、默认羁绊和辅助装备构建证据。
- 支持按样本、可用前四字段、score 和平均名次选择含目标英雄的默认阵容。
- 候选阵容接近且羁绊差异显著时会要求澄清。
- 找不到稳定阵容时可降级为不带默认羁绊的英雄装备查询，并明确提示。

默认上下文仍与全局阵容榜服务职责分离，单英雄装备热路径不需要经过全局榜聚合。

### 3.4 本地化与版本审计

- 当前装备 canonical 简中来自腾讯官网 TFT 装备目录，Riot Data Dragon 英文仅作权威英文回退。
- 当前生成快照覆盖实时 MetaTFT `/items` 179/179，缺失简中 0。
- 刷新脚本校验 TFT patch、source patch、season、版本和源 SHA-256。
- `smoke:item-localization` 会比较实时重建快照与仓库快照，发生漂移时失败。
- patch 审计只报告新增、消失、缺失本地化和名称变化；消失 token 不会自动被判定为已移除。

### 3.5 小窗、缓存与反馈

- 提供本地 HTTP 小窗、固定尺寸 Edge/Chrome app window、置顶定位和全局热键恢复。
- 支持 Memory、JSON 和可选 SQLite cache store；当前系统默认使用 JSON。
- 支持查询缓存、默认阵容缓存、目录快照、偏好、会话、候选别名和反馈事件。
- 热缓存目标 `<=100ms`，本地持久缓存目标 `<=300ms`；最近 smoke 为 1ms/3ms。
- 结果卡支持好/坏反馈，服务端幂等，反馈不会自动改变排序或污染主字典。

### 3.6 可选 LLM

- 默认不启用 LLM，标准英雄装备查询不需要 LLM。
- 项目根目录会自动加载 `.env`，支持：

```env
OPENAI_API_KEY=...
OPENAI_BASE_URL=https://aihubmix.com/v1
MODEL_NAME=...
```

- `OPENAI_BASE_URL` 会规范为 `/chat/completions` 请求地址。
- LLM 当前只输出严格 JSON，并仍需经过本地 schema、实体目录、Context Builder 和 Query Validator。
- endpoint、model 和 API key 不进入浏览器长期偏好；真实 `.env` 不提交到 Git。
- LLM schema 支持单英雄装备、装备可用性和 `comp_rankings`；阵容 metrics/limit 也必须通过本地枚举与范围校验。

### 3.7 阵容排行榜与图标资产

- `exact_units_traits2` 已通过 2026-07-11 实际响应验证；只有完整八桶 `placement_count` 才生成前四率和登顶率。
- 阵容按发布 cluster 或规范化英雄/羁绊指纹聚合，分别提供前四、登顶、平均名次和热门榜。
- 默认排除 PvE、重复英雄、异常人口、召唤物与明确专属强化；低样本仅作为不进入排名的参考。
- Riot Data Dragon `16.13.1` asset manifest 统一解析 unit/item/trait 图标，URL 经过后端 host 白名单；缺图使用固定尺寸文字占位。
- 数据源与风险细节见 `docs/comp-ranking-data-source.md`。

## 4. 当前 API 与数据职责

| 数据 | 来源 | 当前用途 |
|---|---|---|
| 英雄三件套 `placement_count` | MetaTFT Explorer `unit_builds/{unit}` | 计算装备组合样本、前四率、登顶率、平均名次 |
| 阵容八桶 `placement_count` | MetaTFT Explorer `exact_units_traits2` | 阵容聚合、分指标榜单、低样本参考 |
| 当前英雄/羁绊目录 | Explorer 聚合端点 + `/comps` | 实体识别、校验、默认上下文 |
| 当前装备目录 | MetaTFT `/items` | 动态装备 API ID 和统计范围 |
| 阵容 cluster/成员/羁绊 | MetaTFT `/comps` | 单英雄查询的默认阵容上下文 |
| 装备简中 | 腾讯官网 TFT 装备目录 | canonical 中文名 |
| 装备英文 | Riot Data Dragon | 英文别名与缺失简中回退 |
| 英雄/装备/羁绊图标 | Riot Data Dragon 16.13.1 | 版本化 asset manifest 与白名单 CDN URL |

MetaTFT 是非官方接口。实时 smoke 是环境证据，不是稳定 SLA；常规测试必须使用离线 fixture。

## 5. 当前前端状态

当前装备卡显示目标英雄头像和三件装备图标、短名称及 tooltip。阵容榜使用可展开卡：折叠态显示阵容名、主要羁绊、核心/主要成员、榜单指标与样本；展开态显示完整成员、羁绊、明确的核心装备归属、统计和来源。图标与 fallback 均为固定尺寸，460px/360px 使用独立响应式布局。

## 6. 当前明确不支持

以下能力仍不在当前范围：

```text
自动生成完整运营路线、站位图或经济节奏。
从截图/OCR/客户端内存自动识别玩家棋盘。
```

仍不支持 OCR、棋盘自动识别、客户端内存读取、自动操作游戏、完整站位推荐和完整原生悬浮窗能力。没有可靠结构化数据时也不会猜测星级、站位、装备归属或阵容评级。

## 7. 当前阶段验收证据

- `npm test`：187/187 通过，包含阵容增量与原有装备链路全量回归。
- `npm run smoke:comps`：离线 fixture、缓存复用、分指标榜首、零 `unit_builds` 调用及小窗 HTTP 序列化通过。
- `npm run smoke:comps:live`：250 行实时 exact 数据与当前 cluster 定义联合检查通过；最近样本范围 6,446,912。
- `npm run audit:assets`：Riot Data Dragon 16.13.1、662 条 manifest 与固定版本源一致。
- `npm run smoke:small-window`：通过，最近热缓存 4ms、本地缓存 6ms。
- `npm run smoke:item-localization`：179/179 官方简中，快照一致。
- `npm run audit:item-patch`：新增 1、移除 0、缺失本地化 0。
- `npm run audit:items`：当前显式装备可用性规则通过。
- `npm run audit:aliases`：英雄 62/62、羁绊 100/100、装备 169/169。
- 最终版阵容榜已由 bundled Playwright + 本地无头 Edge 实际渲染。460px/360px、正常、缺图、stale、低样本和空结果状态均有完整截图；页面、面板与卡片无横向溢出或紧凑文字裁切，英雄 32px、羁绊 22px、核心装备 18px，缺图占位保持同尺寸。首次空结果检查发现长段位文本会撑宽 grid item，修复后全场景重跑通过。
- `smoke:llm` 已提供，但需要真实 provider key 和 model 才能执行；自动测试已覆盖假 provider 链路。

## 8. 后续产品目标

后续在 patch 更新流程中自动刷新/审计阵容 fixture 与 asset manifest，并把 bundled Playwright 参数固化到发布环境。运营路线、站位或更细粒度阶段策略只有在找到可靠结构化来源后再进入范围。
